# 論文 Methods 記載例（エクスポート画面のコピー用文案）

S10（`#/export`）に表示する「論文の Methods セクションにこう書ける」記載例の正典。エクスポート画面にカードとして表示し、ワンクリックでクリップボードへコピーできるようにする（[requirements.md §4.4](requirements.md)、[ui-states.md `#/export`](ui-states.md)）。

- 記載例は **言語（English / 日本語）× ワークフロー（単一レビュアー検証 / 二重独立検証）** の 4 変種を用意する。MVP のワークフローは単一レビュアー（`ai` 行 + `human_with_ai` 行）なので、既定表示は「単一レビュアー」。二重独立（P1）はデータ構造上は対応済みのため文案だけ先に用意しておく。
- `{{...}}` はプレースホルダ。自動反映できるものはプロジェクトの実績値で埋め、埋められないものはプレースホルダのまま表示してユーザーに編集を促す（§3）。
- 内容は PRISMA 2020 チェックリスト item 9（data collection process: レビュアー数・独立性・自動化ツールの詳細）を満たすことを意図している。

## 1. English

### 1.1 単一レビュアー検証（MVP 既定）

> **Data extraction.** Data were extracted using sr-data-extraction-plugin version {{tool_version}} (https://github.com/youkiti/sr-data-extraction-plugin), an open-source browser extension for large language model (LLM)–assisted data extraction with human verification. A draft data extraction form was generated by an LLM ({{model_id}}, accessed via the {{provider}} API) from the review protocol and {{n_sample}} sample articles, and was reviewed and revised by the authors. The form was piloted on {{n_pilot}} studies and refined before full extraction.
>
> For each included study, the LLM pre-extracted every item from the full-text PDF and returned, for each value, a verbatim quotation of the supporting text, which the tool located and highlighted in the source PDF. One reviewer ({{reviewer_initials}}) checked every AI-extracted value against the highlighted source text and accepted, corrected, or rejected it, or marked the item as not reported. Verification fields were initially blank and every item — including accepted values — required an explicit decision by the reviewer, so no AI-generated value entered the final dataset without human confirmation. All AI outputs (values, quotations, page locations, and confidence ratings) and all human decisions were automatically logged, and the complete audit trail is provided in {{supplement_ref}}.

### 1.2 二重独立検証 + コンセンサス（P1）

2 段落目を以下に差し替える。

> For each included study, the LLM pre-extracted every item from the full-text PDF and returned, for each value, a verbatim quotation of the supporting text, which the tool located and highlighted in the source PDF. Two reviewers ({{reviewer_initials}}) independently verified the AI-extracted values against the highlighted source text, accepting, correcting, or rejecting each value or marking it as not reported. Disagreements were resolved by discussion or, when necessary, by a third reviewer ({{adjudicator_initials}}), and the consensus values were used in the final dataset. Verification fields were initially blank and every item required an explicit decision, so no AI-generated value entered the final dataset without human confirmation. All AI outputs (values, quotations, page locations, and confidence ratings) and all human decisions were automatically logged, and the complete audit trail is provided in {{supplement_ref}}.

### 1.3 オプション文（該当時のみ末尾に追加）

- スキャン PDF（`no_text_layer`）を含むプロジェクト:

  > For {{n_scanned}} studies available only as scanned PDFs without a text layer, the PDF was submitted directly to the LLM; supporting quotations for these studies could not be highlighted and were verified against the quoted text and reported page numbers.

## 2. 日本語

### 2.1 単一レビュアー検証（MVP 既定）

> **データ抽出.** データ抽出には、オープンソースのブラウザ拡張である sr-data-extraction-plugin バージョン {{tool_version}}（https://github.com/youkiti/sr-data-extraction-plugin）を使用した。抽出フォームの草案は、レビュープロトコルとサンプル論文 {{n_sample}} 本をもとに大規模言語モデル（LLM: {{model_id}}、{{provider}} API 経由）が作成し、著者らが内容を確認・修正した。フォームは {{n_pilot}} 本の研究でパイロット抽出を行い、改訂したうえで本抽出に用いた。
>
> 各採用研究について、LLM が全文 PDF から各項目の値を事前抽出し、値ごとに根拠となる本文の逐語的引用を付与した。本ツールは引用箇所を PDF 上に対応付けてハイライト表示し、レビュアー 1 名（{{reviewer_initials}}）がすべての AI 抽出値を原文と照合して、承認・修正・棄却・未報告のいずれかを判定した。検証欄は空欄の状態から開始し、承認を含むすべての項目に明示的な判定操作を必須としたため、人間の確認を経ていない AI 出力が最終データセットに含まれることはない。すべての AI 出力（値・引用・ページ位置・確信度）と人間の判定は自動記録され、監査証跡の全体を {{supplement_ref}} に示す。

### 2.2 二重独立検証 + コンセンサス（P1）

2 段落目を以下に差し替える。

> 各採用研究について、LLM が全文 PDF から各項目の値を事前抽出し、値ごとに根拠となる本文の逐語的引用を付与した。本ツールは引用箇所を PDF 上に対応付けてハイライト表示し、レビュアー 2 名（{{reviewer_initials}}）が独立に、すべての AI 抽出値を原文と照合して承認・修正・棄却・未報告のいずれかを判定した。不一致は協議により解消し、必要に応じて第 3 のレビュアー（{{adjudicator_initials}}）が裁定して、コンセンサス値を最終データセットに用いた。検証欄は空欄の状態から開始し、すべての項目に明示的な判定操作を必須としたため、人間の確認を経ていない AI 出力が最終データセットに含まれることはない。すべての AI 出力（値・引用・ページ位置・確信度）と人間の判定は自動記録され、監査証跡の全体を {{supplement_ref}} に示す。

### 2.3 オプション文（該当時のみ末尾に追加）

- スキャン PDF（`no_text_layer`）を含むプロジェクト:

  > テキスト層を持たないスキャン PDF のみ入手可能であった {{n_scanned}} 本については、PDF を直接 LLM に送信して抽出した。これらの研究では引用箇所のハイライト表示ができないため、引用文と報告ページ番号に基づいて原文と照合した。

## 3. プレースホルダ一覧と自動反映元

| プレースホルダ | 内容 | 自動反映元 |
| --- | --- | --- |
| `{{tool_version}}` | 拡張のバージョン | `chrome.runtime.getManifest().version` |
| `{{model_id}}` | 抽出に使ったモデル | `ExtractionRuns` の `model_version`（`run_type = full` の最新。複数モデル混在時は列挙） |
| `{{provider}}` | API プロバイダ名 | `ExtractionRuns.provider`（`gemini` → `Gemini` 等の表示名変換） |
| `{{n_sample}}` | スキーマドラフトに渡したサンプル論文数 | draft-schema 実行記録（なければプレースホルダのまま） |
| `{{n_pilot}}` | パイロット抽出の対象研究数 | `ExtractionRuns` の `run_type = pilot` の対象 document 数（重複除去） |
| `{{n_scanned}}` | `no_text_layer` の document 数 | `Documents.text_status` の集計。0 件ならオプション文自体を出さない |
| `{{reviewer_initials}}` | レビュアーのイニシャル | 自動反映しない（人間の annotator 数から人数の整合チェックのみ行う） |
| `{{adjudicator_initials}}` | 裁定者のイニシャル | 自動反映しない |
| `{{supplement_ref}}` | audit.csv を載せる補足資料の番号（例: Supplementary Table S1） | 自動反映しない |

- 自動反映できなかったプレースホルダは `{{...}}` のまま表示し、カード内の注意書き（「{{ }} の箇所はご自身の情報に置き換えてください」）で編集を促す。文案の数値・人数・独立性の記述が実際のワークフローと一致しているかはユーザーの責任で確認してもらう。
- 既定 LLM モデルが未確定（Q8）のため、`{{model_id}}` は必ず実行記録から取る（文案にモデル名をハードコードしない）。

## 4. UI 仕様メモ（S10 カード）

- `#/export` の形式選択カードの下に「論文 Methods 記載例」カードを置く。
- 切替: 言語タブ（English / 日本語。既定は English — 投稿論文の主想定言語）× ワークフロートグル（単一レビュアー / 二重独立。既定は単一レビュアー。二重独立は v0.11 で独立二重レビュー機能が実装済みのため両方とも常時有効 — 本文執筆当時「二重は P1 で開放」としていたが実装が追いついたため 2026-07-12 に更新）。
- 本文は読み取り専用の複数行表示 + 「コピー」ボタン（`navigator.clipboard.writeText`）。成功時は「コピーしました」フィードバック（tiab-review のトースト トンマナ）。
- スキャン PDF のオプション文は `{{n_scanned}} > 0` のときだけ本文に連結する。
- コピーされるのは引用ブロック内のプレーンテキスト（見出し `**Data extraction.**` は Markdown 太字のまま含めず、`Data extraction.` として先頭に付ける — 投稿システムはプレーンテキスト前提）。
