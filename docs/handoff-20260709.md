## 1. このフェーズで何をしたか（3 行で）

SR では 1 つの試験（study）が複数の PDF（本論文・試験登録・プロトコル論文・学会抄録）で報告される。フェーズ 1 で study / document を分離したうえで、本フェーズは **AI 抽出を「1 API 呼び出し = 1 study」に変更**した。study の全文書をロール付き区切りで連結して 1 回で抽出し、応答の `document_index` で各 quote の出所文書を解決して `Evidence.document_id` に記録する。パイロット（S6）・一括抽出（S7）の対象選択・進捗も document 単位から study 単位へ移行した。

## 3. 設計判断（実装の根拠。フェーズ 3 で壊さないこと）

1. **`Evidence` は study_id + document_id の 2 キー**。study_id が抽出単位、document_id が quote の出所文書。アンカリング・ハイライトは document_id の文書に対して行う（§5）
2. **document_index の解決規則**（`validateAiOutput`）:
   - quote あり + 欠落/範囲外 → 破棄（`partial_failure`）。**ただし文書 1 件のみの study なら欠落を index=1 へ寛容に解決**（出所が一意で誤帰属の余地がないため。requirements の「誤帰属防止」意図を破らない）
   - `not_reported`（quote なし）→ document_index 不要、index=1（先頭 = 主文書）へ帰属
3. **部分ロード失敗の続行**: study の一部文書だけロードできれば、ロードできた文書だけを連結して抽出を続ける（document_index はロード済み文書の並びで詰め直す）。全滅時のみ `load_failed`
4. **抽出済み判定は完了 run の study_ids**（`ExtractionRuns` の 2 行プロトコルは不変更）。中断 run の study は未抽出扱いに戻り S7 の既定選択で再開できる
5. **プロンプト版数を必ず上げた**（1 → 2）。プロンプト文言・スキーマを変えたら `EXTRACT_DATA_PROMPT_VERSION` をインクリメントする規約

## 5. フェーズ 3（次の担当者へ）

**ゴール**: `#/verify?study=` で study を開き、根拠クリックで出所 PDF へ自動切替。ダッシュボード・CSV も study 単位。詳細は [handoff-study-document-model.md](handoff-study-document-model.md) §フェーズ 3。要点のみ：

- `app/router.ts` / `bootstrap.ts`: `docQueryOf` → `studyQueryOf`（`?doc=` → `?study=`）
- `verifyService.ts` / `verificationService.ts`: 検証データ束を study 単位で（複数 PDF + 各文書のテキスト層。**PDF バイナリは文書切替時の遅延読込** = §6 性能）
- `app/ui/pdfViewer.ts` / `verificationPanel.ts`: **文書切替タブ**（role バッジ + filename）。項目フォーカス / ハイライトジャンプ時に `Evidence.document_id` の文書へ自動切替してからスクロール。描画競合の連番ガードを文書切替でも維持
- `dashboard.ts` / `dashboardService.ts` / `dashboardView.ts`: マトリクスを study × section へ
- `buildStudyWideCsv.ts` / `buildResultsLongCsv.ts` / `buildAuditCsv.ts`: study_label / study_id は Studies 由来。audit.csv に document_id 列。非アクティブ study は出力しない
- E2E は最小実 PDF stub を **2 本**配信する形へ拡張

**フェーズ 2 で残した暫定挙動（フェーズ 3 で解消）**: パイロット / S8 の埋め込み検証 UI は現状 study の代表 1 文書（article）しか開かないため、別文書由来の quote はハイライトされない（Evidence の document_id は正しく記録されている）。フェーズ 3 の複数文書ビューアで解消する。
